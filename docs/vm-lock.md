# 虚拟机锁定功能

## 概述

虚拟机锁定功能用于保护重要虚拟机不被误删除或误关机。锁定后的虚拟机无法被手动关机和删除，解除锁定时需要进行二次验证（邮箱验证码或 2FA）。

## 功能说明

### 锁定后的限制

- **关机/强制断电需二次确认**：锁定的虚拟机仍可关机和强制断电，但会弹出二次确认弹窗，提示用户操作可能影响正在运行的服务
- **禁止删除**：无法通过面板删除虚拟机（包括定时删除任务）
- **允许开机**：锁定的关机虚拟机仍然可以开机
- **允许重启**：锁定的运行中虚拟机仍然可以重启

> **注意**：锁定不影响系统级操作，如维护模式全量关机、运行时长配额耗尽自动关机等。

### 锁定/解锁权限

- **弹性云用户**：可以锁定和解锁自己拥有的虚拟机
- **管理员**：可以锁定和解锁所有虚拟机
- **轻量云用户**：无权使用锁定功能

## API 接口

### 锁定虚拟机

```
POST /api/vm/:name/lock
```

- **权限**：弹性云用户或管理员
- **无需二次验证**
- **响应示例**：

```json
{
  "code": 200,
  "message": "虚拟机已锁定"
}
```

### 解锁虚拟机

```
POST /api/vm/:name/unlock
```

- **权限**：弹性云用户或管理员
- **需要二次验证**（428 状态码 → TOTP/邮箱验证）
- **响应示例**：

```json
{
  "code": 200,
  "message": "虚拟机已解锁"
}
```

### 获取锁定状态

```
GET /api/vm/:name/lock
```

- **权限**：已登录用户（含轻量云）
- **无需二次验证**
- **响应示例**：

```json
{
  "code": 200,
  "message": "ok",
  "data": {
    "id": 1,
    "vm_name": "test-vm",
    "locked": true,
    "locked_at": "2026-05-12T10:30:00Z",
    "locked_by": "admin"
  }
}
```

### VM 列表/详情中已包含锁定状态

在 VM 列表 (`GET /api/vm/list`) 和 VM 详情 (`GET /api/vm/:name`) 的返回数据中，已新增 `locked` 字段（布尔值），前端可直接使用。

## API Key 支持

所有锁定相关接口默认兼容 API Key 调用。解锁操作的二次验证同样适用于 API Key 请求。

## 数据库

锁定状态存储在 `vm_locks` 表中：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| vm_name | TEXT | 虚拟机名称（唯一索引） |
| locked | BOOLEAN | 是否锁定 |
| locked_at | DATETIME | 锁定时间 |
| locked_by | TEXT | 锁定操作者用户名 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

虚拟机删除时，关联的锁定记录会自动清理。

## 前端界面

### 虚拟机列表页

- "更多"下拉菜单中新增「锁定虚拟机」/「解除锁定」选项
- 锁定状态下：
  - 关机/强制断电按钮仍可点击，但会弹出二次确认弹窗，提示"虚拟机已锁定，关机可能影响正在运行的服务"
  - "删除"选项变灰且不可点击

### 虚拟机详情页

- Hero 横幅区域显示橙色"已锁定"标签
- 关机按钮变为橙色"关机（已锁定）"，点击后弹出二次确认弹窗
- 断电按钮变为红色"强制断电（已锁定）"，点击后弹出二次确认弹窗
- 额外提供「锁定虚拟机」/「解除锁定」快捷按钮
