# 直通 GPU 作为主显示设备

## 使用场景

Windows 虚拟机直通独立显卡后，如果设备管理器提示代码 31，并显示“尝试启动的驱动程序与 POSTed 显示适配器的驱动程序不同”，通常是虚拟 VGA 仍被固件作为主显示设备。

## 配置方法

1. 正常关闭虚拟机。
2. 在虚拟机高级设置中，将“显示设备”选择为 `None（禁用虚拟显示）`。
3. 在硬件直通中添加显卡及其 IOMMU 组内需要一并直通的功能，例如 NVIDIA HDMI 音频设备。
4. 保存配置并冷启动虚拟机。

当显示设备为 `none` 且仅检测到一张直通 PCI VGA 时，后端会自动为该显卡设置 `x-vga=true`。`none` 也可以在没有直通显卡时单独用于无头虚拟机。

## 控制台变化

禁用虚拟显示设备后，虚拟机详情页会隐藏 VNC 和 SPICE 控制台。此时可使用以下方式管理虚拟机：

- QEMU Guest Agent；
- 虚拟机内安装的远程桌面或串流软件；
- 直通显卡的物理视频输出。

直接访问旧的 VNC 独立窗口时，页面会提示虚拟显示设备已经禁用，不会尝试建立连接。

## 限制与排障

- 修改显示设备和主显卡关系必须先关机，运行时热修改不能重新执行固件 POST。
- 同一虚拟机存在多张直通 VGA 时，系统不会猜测主显卡，需要先仅保留一张 VGA 后再启用 `none`。
- 系统不会自动安装 NVIDIA、AMD 或 Intel 的来宾驱动。若显卡已经成为主显示设备但驱动仍未加载，应在虚拟机内部重新安装或绑定对应驱动。
- VNC/SPICE 的图形协议配置可以保留，但不会自动补回 QXL、VGA 等虚拟显示设备。

## API

创建、克隆、导入或编辑虚拟机时，可传递：

```json
{
  "video_model": "none"
}
```

虚拟机详情接口会返回 `video_model: "none"`，前端据此隐藏 VNC 和 SPICE 控制台入口。

单台模板克隆支持同时传入 `host_devices`；批量克隆不能把同一物理 PCI 设备分配给多台虚拟机，因此 `count > 1` 时会拒绝 `host_devices`。
